# Security Policy

## Supported Versions

The latest version on the `master` branch is actively supported.
Older snapshots, forks, and local modifications are not guaranteed to receive security fixes.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities in public issues.

Use one of these private channels:

1. GitHub Security Advisories (preferred):
   - Open the repository on GitHub.
   - Go to `Security` -> `Advisories` -> `Report a vulnerability`.
2. If advisory reporting is unavailable, contact the maintainer directly via a private channel.

## What to Include

To help triage quickly, include:

- A clear description of the vulnerability.
- Steps to reproduce.
- Impact assessment (what could be abused).
- Affected files/endpoints and commit/branch.
- Suggested mitigation (if available).

## Response Timeline

- Initial acknowledgement: within 72 hours.
- Triage decision: within 7 days.
- Fix timeline depends on severity and complexity.

## Disclosure Policy

Please allow time for investigation and patching before any public disclosure.
Once a fix is available, coordinated disclosure is welcome.

## Scope Notes

This policy covers code in this repository.
It does not cover third-party services or dependencies outside the repository's control.
